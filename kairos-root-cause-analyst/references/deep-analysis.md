# 深度根因分析工作流

前置条件：已完成轻量三步（定性 / 根因假说 / 验证建议），且问题复杂、
多方利益纠缠或无法收敛。按需选用以下展开，不必全跑。

## 展开一：诚实性校准

只在用户的表述与其真实目标可能不一致时插入（例如用户在自我辩护、
回避责任、或问题定义本身可疑）：

- 目标函数诚实：确定在优化正确的东西，还是"看起来合理"的东西？
- 利益诚实：有没有没被说出口的利益驱动？
- 能力边界诚实：哪些是"还没去了解"，哪些是"目前没人知道"？

## 展开二：三路径推演

根据定性结果选路径，至少走两条交叉验证（问题足够简单除外）：

| 问题类型 | 优先路径 |
|---|---|
| 认知问题 | 第一性原理为主，矛盾论辅助 |
| 利益问题 | 利益分析为主，矛盾论辅助 |
| 结构性/系统性问题 | 矛盾论为主 |

### 矛盾论（结构视角）

列出对立统一的力量 → 找主要矛盾 → 找矛盾的主导方面 →
推演：若主导方面改变，系统会怎样。
局限：定性不定量。

### 第一性原理（要素视角）

区分事实与假设 → 逐条质疑假设（去掉它问题还在吗）→
拆到不可再拆的要素 → 不因"别人都这么做"而接受任何前提。
局限：可能忽略路径依赖。

### 利益分析（动力视角）

- Cui bono（谁受益）：现状必有受益者，找到他们就找到维持现状的力量
- Cui malo（谁受损）：变革动力来自受损方，无受损方则无变革动力
- Quis iudicat（谁裁判）：决策者与后果承担者是否同一人，
  不是则存在代理成本

局限：人会为意义、尊严、归属感做反利益行为。

## 展开三：三角定位

- 路径结论一致 → 置信度高，进入求解
- 两者一致、一者不同 → 那条不同的路径可能指向盲区，想清楚为什么
- 三者都不同，或问题涉及纯情感/审美判断 → 诚实告知用户
  "尚未触达本质"，建议做实验获取信息，而非继续推理

## 展开四：求解

### 条件盘点

- 已有条件：是否被充分利用？
- 可创造条件：创造成本 vs 收益？
- 不可改变约束：确认真的不可改变

### 约束检验

遇到"没办法"，追问：违反它会有什么客观上必然发生的后果？
如果答案是"不太好""没人这么做过"，那是自我约束，不是硬约束。
多数人 80% 精力用于抱怨硬约束、20% 用于挖掘已有条件——帮用户反过来。

### 综合收益（内心权衡，不向用户展示公式）

问题解决度、可复用性、学习收益、关系收益，
减去机会成本和路径锁定风险。
主动指出用户可能忽略的收益或代价。

### 最小验证实验

- 能在 1 天内跑完
- 检验核心假设而非边角细节
- 有明确的通过/失败标准
- 失败不伤筋动骨
