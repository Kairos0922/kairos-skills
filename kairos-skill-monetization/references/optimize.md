# 阶段二：价值优化

前置假设：已验证值得做。任务：同样的核心能力，如何用最小边际成本换最大愿付价格和覆盖规模。若评估阶段在本会话完成，直接沿用已知信息，不重复提问。

## 1. 现状定位（若信息缺失才问）

状态（自用/1-2客户/稳定收入/规模化瓶颈）× 人工介入程度 × 本次目标（客单价/成本/规模/壁垒）。
未验证付费的不谈规模化；已有客户但全手工的，优先杠杆化而非价值方程。

## 2. 价值方程（提升愿付价格）

感知价值 = (梦想结果 × 确定性) / (时间延迟 × 所需努力)

- 梦想结果：功能语言 → 结果语言（"帮你整理笔记" → "每周省 3 小时，多产出 2 篇"）
- 确定性：案例/演示/担保/社会证明缺哪个？
- 时间延迟：能否设计"首次使用几分钟内的即时小胜利"？
- 努力：每减一个操作步骤/学习门槛，愿付价格上升
产出：指出最拖后腿的变量 + 2-3 个具体改造动作。

## 3. 杠杆化（降低边际成本）

| 杠杆 | 判断问题 |
|---|---|
| 劳动力 | 哪些步骤依赖你亲自做？ |
| 资本 | 该花钱买效率还是先用代码解决？ |
| 代码 | 核心逻辑能否变成模板/自动化/产品界面？ |
| 媒体 | 交付过程能否内容化（教程/案例），反哺获客？ |
优先级：靠本人时间的环节 → 先问能否变代码，再问能否变媒体（SOP 化）。代码+媒体可叠加。

## 4. 价值阶梯（覆盖不同付费层）

- 入门层：低价/免费，获客建信任
- 核心层：主力收入，已验证的核心价值
- 高端层：人工介入/定制/结果保证，收确定性溢价
追问：现在只覆盖了哪层？只做核心层 = 丢掉长尾和溢价两头。

## 5. 护城河加固

- 数据壁垒：每次使用能否积累专属数据/案例库？
- 工作流绑定：能否嵌入日常工作流而非"想起来才用"？
- 品牌信任：持续内容输出成为细分问题的默认答案
- 网络效应：用户之间能否互相产生价值？（最难复制）
诚实检查：本质无法建壁垒的（纯公开 prompt 技巧），转向"抢速度 + 先发品牌认知"，不虚构护城河。

## 6. 验证闭环

每个优化动作配 1-2 周可观察的数字指标（愿付价格/转化率/交付时长/复购率），不用"客户是否满意"。
定价按价值锚点（客户不用要付出什么代价），禁止成本加成。涨价是最被低估的杠杆——用一小批新客户测试更高价位。
