# Product — kairos-visual-generator

这份文档讲**为什么**。事实在 `SKILL.md`，设计规范在各风格的 `DESIGN.md` 中。

---

## 1. 这个 skill 解决什么问题

**真正的问题不是"画图"，是"让非设计师也能产出多种风格的高质量视觉卡片"。**

普通用户的现实链路：

1. 写完一篇文章或做完一份报告。
2. 想做一张封面、信息图或海报发到社交媒体。
3. 没有设计师，Canva 模板太千篇一律。
4. 让 AI 生成，结果大部分是"标题放正中 + 渐变背景 + emoji"。

我们在第 3-4 步之间插一个东西，让产出**有灵魂、有隐喻、有克制**。

### 核心矛盾

- **质量**：要像顶级咨询报告封面或 *Monocle* 杂志卡片，不能像 PPT 模板。
- **效率**：要 10 分钟内拿到可发布的图，不能让用户在 prompt 上反复打磨。
- **不假**：数据、报价、版本号不能编。

三者之中**质量是分水岭**。Canva 解决了效率，AI 解决了速度，但都没解决"看一眼就知道是用心做的"这件事。

---

## 2. 产品定位（一句话）

> 把一个主题交给 AI agent，10 分钟内输出一张有隐喻、有克制、有风格的视觉卡片或海报。

---

## 3. 目标用户

| 场景 | 用户类型 | 输入 | 期望输出 |
| --- | --- | --- | --- |
| X / 公众号封面 | 创业者 / DevRel | 主题 + 用途 | 封面卡片 |
| 小红书封面 | 内容创作者 | 主题 + 画幅 | 3:4 卡片 |
| 商业报告封面 | 咨询顾问 | 主题 + 语境 | 16:9 封面 |
| 信息图 | 产品经理 | 主题 + 数据 | 结构卡片 |
| 方法论图 | 研究员 | 主题 + 框架 | Swiss 风信息图 |
| 海报 | 设计师 / 创意 | 主题 + 风格 | Mondrian 风海报 |

**反向画像**：

- 想做卡通 IP 风、少女风、Y2K：和三套视觉系统正面冲突，硬接出来一定难看。
- 想做复杂数据大屏：这不是数据可视化工具。
- 想自定义颜色：不允许，保护美学比给自由更重要。

---

## 4. 关键设计决策

### 决策 1：三套视觉系统，风格即插件

- **Editorial Magazine**：serif + 墨水 + 杂志结构
- **Swiss Consulting**：Inter + 一种 accent + 工程结构
- **Mondrian / De Stijl**：五原色 + 色块网格 + 文字即结构

每套风格是独立的插件，有自己的 DESIGN.md、composition.json 和主题 token。新增风格 = 新增目录，不改核心代码。

### 决策 2：不允许自定义颜色

从每套风格的主题预设中选择。让用户传一个 `#A3FF21` 进来，AI 配的衍生色大概率难看。

### 决策 3：质量不确定时必须追问

风格错误 = 质量归零。宁可多问一个问题，也不猜测。

### 决策 4：单一核心隐喻

一张卡片只允许一个主隐喻。多个隐喻同时堆叠会让画面失控。

### 决策 5：标题必须信息重构

长主题必须拆成三层：A 层核心词、B 层完整标题、C 层极简标签。主视觉文字不能只是普通标题，必须成为画面主体结构。

### 决策 6：确定性通过代码实现

AI 生成 Visual Brief，代码消费 Brief 输出 PNG。渲染管线是纯确定性的，AI 不参与。

---

## 5. 视觉系统锁定

三套视觉系统的本质不可改变：

| 系统 | 视觉本质 | 禁止改变 |
| --- | --- | --- |
| Editorial Magazine | 衬线 + 墨水 + 杂志结构 + 大留白 | 不加网格、不加数据大字报、不加霓虹色 |
| Swiss Consulting | Inter + 一种 accent + 网格 + 发丝线 | 不加衬线装饰、不加水墨效果、不加渐变 |
| Mondrian / De Stijl | 五原色 + 色块网格 + 文字即结构 | 不加渐变、不加复杂插画、不加非原色 |

---

## 6. 质感规则

输出必须像高级印刷品，不是 PPT 模板。

**必须**：平涂色块、线条清晰、留白充足、颜色克制、文字清晰

**禁止**：渐变、霓虹色、玻璃拟态、复杂阴影、油腻光效、卡通质感、商务小人

---

## 7. 产品边界

| 不做 | 原因 |
| --- | --- |
| 不允许自定义颜色 | 保护美学一致性 |
| 不做卡通/少女/Y2K 风格 | 和三套视觉系统冲突 |
| 不做复杂数据大屏 | 这是卡片，不是 dashboard |
| 不在 skill 内调用生图模型 | 生图是宿主 agent 的能力 |
| 不允许多个隐喻堆叠 | 单一隐喻是产品承诺 |

---

## 8. 反模式

1. **普通大标题**：没有文字重构的标题 → 拒绝。
2. **没有视觉隐喻**：标题旁边放图标冒充图文融合 → 拒绝。
3. **商务小人 / 握手图标 / 机器人头像** → 拒绝。
4. **科幻蓝紫背景 / 霓虹渐变** → 拒绝。
5. **5 个以上模块堆叠**：信息图失控 → 拒绝。
6. **假数据 / 假版本号 / 假报价** → 拒绝。
7. **文字被遮盖或裁切** → 拒绝。
8. **自媒体廉价感** → 拒绝。
