# Anti-patterns

全局反模式清单。每套风格还有自己的反模式，见各风格的 `DESIGN.md`。

## 绝对禁止（所有风格）

1. **普通大标题**：没有文字重构的标题 → 拒绝
2. **图标冒充图文融合**：标题旁边放图标 → 拒绝
3. **商务小人 / 握手图标 / 机器人头像** → 拒绝
4. **科幻蓝紫背景 / 霓虹渐变** → 拒绝
5. **5 个以上模块堆叠**：信息图失控 → 拒绝
6. **假数据 / 假版本号 / 假报价** → 拒绝
7. **文字被遮盖或裁切** → 拒绝
8. **自媒体廉价感** → 拒绝
9. **渐变 / 霓虹色 / 玻璃拟态** → 拒绝
10. **复杂阴影 / 油腻光效** → 拒绝
11. **卡通质感 / 儿童感** → 拒绝
12. **PPT 模板风格** → 拒绝
13. **多个隐喻同时堆叠** → 拒绝

## Ticket / Receipt 风格专项反模式

14. **假票据**：有票据外壳但内容空洞 → 拒绝
15. **假条形码**：编码假编号、比例错误、面积过大（>15% 画布） → 拒绝
16. **使用 QR 码**：视觉密度冲突 + 暗示可扫描 = 假元素 → 拒绝
17. **仿冒商业条码格式**：UPC/EAN 等真实条码格式用于虚构数据 → 拒绝
18. **信息过载**：超过 8 个核心字段 → 拒绝
19. **非等宽字体**：核心数据字段使用非等宽字体 → 拒绝
20. **数据不对齐**：key: value 字段没有对齐 → 拒绝

## Bad/Fix 对照

| Bad | Fix |
|-----|-----|
| 标题放在正中 + 渐变背景 | 文字重构 + 锁定色板 + 留白 |
| 标题旁边放一个图标 | 图形从文字结构中生长出来 |
| 5 个模块平均排列 | 选择 2-3 个核心模块，建立层级 |
| 背景放抽象图形但和文字无关 | 图形必须服务标题，不可喧宾夺主 |
| 文字和视觉隐喻互不影响 | 隐喻必须嵌入文字内部或边界 |
| 颜色超过 3 种 | 只用一套预设，accent 色不超过 8% |
| 票据风格用非等宽字体 | 核心数据字段必须使用等宽字体 |
| 票据风格数据不对齐 | 利用等宽字体实现自然对齐 |
| 票据风格条形码面积过大 | 条形码面积 ≤ 8% 画布，位于底部/边缘 |
