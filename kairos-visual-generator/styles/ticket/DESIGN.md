# Ticket / Receipt Design System

## 风格定位

通过票据/收据的视觉语言，将结构化数据转化为有质感的信息 artifact。特征是等宽字体、虚线分隔、打孔标记、对齐的数据字段、票据纸质感。

适合：行程安排、活动日程、清单对比、预算分解、时间轴、发票/收据式数据展示。

## 视觉 DNA

- **色彩哲学**: 票据纸 + 油墨。以暖白/米色为纸底，深色文字，单一 accent（可选）。
- **结构元素**: 等宽字体、虚线分隔、打孔标记、对齐的数据字段、条形码装饰区。
- **质感**: 热敏打印纸、邮票边缘、复古收据。不是"高端印刷品"，而是"有温度的数据 artifact"。
- **信息密度**: 中高。结构化数据为主，字段对齐是核心视觉语言。

## 色彩系统

以票据纸和油墨为基调：

| 语义角色 | 说明 | 允许范围 |
|----------|------|----------|
| paper | 票据纸底色 | 暖白/米色系 |
| ink | 主文字色 | 深色，高对比 |
| muted | 辅助信息色 | 灰色 |
| accent | 强调色（可选） | 克制使用，≤5% |
| line | 分隔线色 | 虚线/实线 |

## 字体配方

使用 Ticket Monospace 配方：

```css
:root {
  --font-mono-en: "Departure Mono", "JetBrains Mono", "Fira Code", monospace;
  --font-mono-zh: "Noto Sans Mono CJK SC", "Noto Sans Mono", "Source Han Sans SC", monospace;
  --font-text-zh: "Noto Sans SC", "PingFang SC", sans-serif;
  --font-en: "Inter", "Helvetica Neue", sans-serif;
}
```

规则：
- 核心数据字段必须使用等宽字体
- 英文数字用 Departure Mono
- 中文用 Noto Sans Mono CJK SC
- 标注/说明文字可用普通无衬线（Noto Sans SC）
- Departure Mono 仅有 Regular 字重，层级区分靠字号和颜色，不靠粗细
- Noto Sans Mono CJK SC 有 Regular + Bold，可用于标题/正文层级
- 启用 `-webkit-font-smoothing: antialiased` 平滑像素字体边缘
- 利用 Box drawing characters 构建结构化布局（替代部分 CSS border）

## 网格系统

- 票据式纵向布局
- 数据字段左对齐或右对齐（等宽自然对齐）
- 虚线分隔区域
- 外边距 48-64px（比其他风格窄，模拟票据边缘）
- 打孔标记在顶部或左侧

## 文字重构

Ticket 的文字重构以数据结构为主：
- 文字成为数据字段的核心载体
- 通过等宽字体实现自然对齐
- 字段 key: value 对齐是核心视觉语言
- 标题区域可以用大号等宽数字或文字

## 票据特有元素

| 元素 | 说明 | 用途 |
|------|------|------|
| 打孔标记 | 顶部/左侧的圆形缺口 | 分割头部区域 |
| 虚线分隔 | dashed border | 区域分隔 |
| 条形码 | 底部结构装饰（见反模式规则） | 底部收尾，编码真实编号 |
| 字段对齐 | 等宽字体自然对齐 | 数据展示核心 |
| 编号/日期 | 元数据标注 | 头部信息 |
| 剪切线 | 边缘锯齿 | 可选装饰 |
| Box drawing chars | ░┌─┬┐│└┴┘├─┼┤ 等 | 结构化表格/边框（可选） |

## 质感规则

**必须**：
- 等宽字体
- 虚线或实线分隔
- 数据字段对齐
- 票据纸质感（暖白/米色底）
- 信息结构清晰

**禁止**：
- 假条形码/QR码（纯装饰无意义）
- 假印章/假签名
- 渐变/霓虹色
- 复杂阴影/玻璃拟态
- 卡通质感
- 假数据

## 反模式

1. 做成"假票据"——有票据外壳但内容空洞
2. 条形码纯装饰——编码假编号、比例错误、面积过大（>15% 画布）
3. 使用 QR 码——视觉密度冲突 + 暗示可扫描 = 假元素
4. 仿冒商业条码格式—— UPC/EAN 等真实条码格式用于虚构数据
5. 信息过载——超过 8 个核心字段
6. 和 Editorial/Swiss/Mondrian 风格混搭
7. 假数据/假编号/假日期
8. 渐变/霓虹/玻璃拟态

## 专项 QA 规则

- 等宽字体是否正确加载
- 虚线分隔是否清晰
- 数据字段是否对齐
- 打孔标记是否自然
- 整体是否有票据感而非 PPT 模板感
- 中英文混排是否协调
- 条形码（如存在）是否编码真实数据
