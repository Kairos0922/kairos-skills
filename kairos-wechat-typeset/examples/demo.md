# 一篇适合公众号的 Markdown 文章

这是开场段。它应该简洁、稳定，而且在复制到微信公众号编辑器之后，依然保持舒服的行高与黑白灰层次。到了 2026 年，像 AI Agent、Markdown、HTML 这样的英文与数字，也应该和正文里的宋体自然地站在一起。==如果一句话值得读者停一下，它就应该被轻轻地强调出来。==

## 01 为什么要先解决兼容性

很多排版工具在浏览器里很好看，但一进微信编辑器就会失真。真正稳定的方案，往往不是“最炫”的，而是最克制的。

> [!NOTE]
> 微信编辑器会过滤很多标签和样式，所以版式设计一定要先考虑保留率，而不是先考虑花样。

- 所有样式尽量写成内联样式
- 结构尽量简单，避免无效嵌套
- 列表、引用、强调都要优先选择微信更容易保留的写法

## 02 什么叫做稳定的层次

**真正好的排版，不是元素越多越高级，而是层次越清楚越省心。**

我们可以用三个动作建立稳定层次：

1. 用标题把结构切开
2. 用留白把阅读节奏放慢
3. 用有限的强调样式，把注意力收束到关键句

> [!WARNING]
> 如果一篇文章同时使用太多颜色、边框和装饰，读者不会觉得丰富，只会觉得吵。

> 引用块适合承接解释、备注和提醒。它不应该比正文更抢眼，但应该比正文更容易被扫到。

```python
def keep_it_simple(article):
    return article.with_inline_styles().prefer_paragraph_blocks()
```

| 方式 | 结果 |
| --- | --- |
| 花哨容器 | 容易在微信里失真 |
| 克制内联样式 | 更稳定 |

最后一段可以收束全文。读者带走的不应该只是“这篇文章排得还不错”，而应该是“我知道以后怎样排版，复制进微信也不容易乱”。
