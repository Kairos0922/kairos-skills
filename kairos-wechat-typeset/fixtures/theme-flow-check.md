# AI 产品发布复盘：从能力到体验

过去一年，AI 产品的竞争从“模型能力”逐渐转向“可被普通用户感知的体验”。一个功能是否先进，已经不能只看 benchmark，而要看它是否能被自然地纳入日常工作。

==真正影响留存的，往往不是一次惊艳演示，而是用户第二天还愿不愿意打开。==

## 01 入口：让复杂能力变得可接近

好的入口不是把全部能力铺开，而是给用户一个明确的第一步。尤其在工具类产品里，过多按钮会稀释判断，过少反馈又会让人不确定系统是否理解了自己。

> [!NOTE] 入口设计要服务于心智建模
> 用户第一次使用时，最需要的不是完整说明，而是一个能立刻成功的动作。

- 明确主任务，减少并列入口。
- 用示例承接陌生概念。
- 在结果之后再提供高级设置。

## 02 反馈：把系统状态说清楚

AI 系统的状态并不总是线性的：检索、推理、生成、校验可能同时发生。界面需要给出足够明确的过程反馈，但不应该把内部机制全部暴露给用户。

| 阶段 | 用户关心 | 界面反馈 |
| --- | --- | --- |
| 输入 | 是否理解意图 | 摘要与可编辑目标 |
| 生成 | 是否仍在工作 | 稳定进度与部分预览 |
| 完成 | 是否可信 | 引用、差异和下一步 |

## 03 质量：让输出能够被接手

最终结果要能进入用户原本的工作流。对于内容工具，这意味着结构清晰、重点可见、引用可靠；对于数据工具，则意味着字段稳定、错误可查、结果可复用。

> [!TIP] 可以把“可接手”当成质量标准
> 如果用户还需要大面积重排、重写或重新核对，产品就只是制造了新的工作。

```python
def handoff_ready(result):
    return result.is_structured and result.has_context and result.next_action
```
