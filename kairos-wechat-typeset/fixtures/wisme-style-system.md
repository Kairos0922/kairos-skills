# 微信公众号文章组件==规范==

:::lead
适用于科普类、知识类、研究报告类文章的排版规范与样式指南。
:::

## 01 文本样式

### H1 标题

这是一级标题对应的正文说明，用于承接文章最重要的信息。标题应该清晰、直接，避免在正文里重复制造封面式装饰。

### H2 标题

二级标题用于划分主要结构。WISME 主题使用编号、黑色标题和细灰线建立规格感。

### H3 标题

三级标题用于段落内部的小结，字号不夸张，但要比正文更明确。

正文内容用于承载主要信息，建议行宽保持克制，段落之间留出 16px 到 24px 的间距，让阅读节奏清楚。

## 03 列表

无序列表：

- 这是无序列表项，用于并列或补充说明。
- 每一项内容简洁清晰，层级分明。
- 建议每行不超过两行，提升阅读体验。

有序列表：

1. 这是有序列表项，用于步骤或流程说明。
2. 每一步都说明了，确保可操作性。
3. 支持多层级嵌套，结构清晰。

任务列表：

- [x] 这是已完成的任务项
- [ ] 这是未完成的任务项

## 04 代码

```python
def reconstruct(memory_trace):
    context = analyze_context()
    cues = retrieve_cues(context)
    memory = integrate(memory_trace, cues)
    return memory.reconstruct()
```

```bash
$ wisme trace --source paper.pdf --output report.md
# 解析完成，生成报告 report.md
```

命令行中可以使用 `wisme trace --source paper.pdf` 作为示例命令。

## 05 表格

| 概念 | 定义 | 关键特征 | 应用场景 |
| --- | --- | --- | --- |
| Reconstructive Memory | 重构性记忆 | 主动重建而非回放 | 教育、治疗 |
| Semantic Drift | 语义漂移 | 概念在传递中产生变化 | 知识传播、AI |
| Epistemic Humility | 认知谦逊 | 承认记忆的不确定性 | 决策、协作 |
| Hippocampal Indexing | 海马索引 | 记忆的时空定位机制 | 神经科学 |
