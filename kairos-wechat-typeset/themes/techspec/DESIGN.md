# 科技主题 DESIGN.md

## Theme Position

科技主题根据附件《微信公众号文章组件规范》落地。它面向 AI 技术文章、工程实践、产品方案、研发规范和工具教程，把参考图中的移动端科技杂志感翻译为微信公众号正文可控表面：强黑标题、蓝色编号、细线秩序、清晰表格、暗色代码块、浅蓝信息提示和稳定的 390px 手机阅读节奏。

本主题不复刻平台 UI。参考图里的手机状态栏、账号头部、底部工具栏和右侧说明页属于设计稿语境；主题只实现正文中的标题、导语、正文、图片、章节、引用、列表、任务、代码、表格、分隔线和信息块。

## Visual Language

- 关键词：科技蓝、纯白纸面、强标题、细线、8px 圆角、暗色代码、浅蓝提示。
- 适用文章类型：AI 技术文章、工程实践、产品方案、研发规范、工具教程。
- 禁止：渐变背景、强投影、多色强调、厚重卡片、营销按钮、深色整页、装饰性图标堆叠。
- 页面气质：像一份已经排进公众号手机正文里的组件规范，清晰、锐利、可信。

## Tokens

参考图里的字号与层级是硬约束：

- H1: 28px / 1.42，黑色，粗体，左对齐。
- H2 numeric: 32px 蓝色编号 + 22px 黑色章节标题，底部蓝色短线与浅灰长线。
- H3: 18px / 1.6，黑色，粗体。
- 正文: 16px / 1.75，`#374151`。
- 辅助信息、列表、图注、代码标签: 14px，灰色或蓝色。
- Code: 14px 等宽字体，暗色面板，8px 圆角，保留行号。

主色只有 `#0b6dff`。浅蓝只作为同一主色的低浓度信息底，不引入第二强调色。

## Components

- H1: 参考手机正文标题，28px 强黑标题，紧凑但不压迫。
- Lead: 作为平台元信息/导语的正文内表达，灰字、小节奏，不做大封面。
- H2 numeric: 蓝色 `01` 数字独立成行，标题在下，下面是一条蓝色短线接浅灰长线。
- H2 non-numeric: 左侧蓝色编号感小标与黑色标题，用于规格类小节。
- H3-H6: 18px 黑色小标题。
- Paragraph: 16px / 1.75，段距 18px。
- Highlight: 浅蓝底、蓝字，用于重点句，不能高频使用。
- Quote / Pullquote: 左侧蓝线，正文灰黑，保留参考图的克制技术引用感。
- Insight / Closing note: 浅蓝信息块，8px 圆角，蓝色左侧符号感由文字和细线承担，不使用图标。
- Figure: 图片 100% 宽，8px 圆角，图注 14px 灰色左对齐。
- Lists: 蓝色圆点或数字，任务清单使用 12px 方框，完成态蓝底白勾。
- Code: 暗色代码块，14px 等宽字体，行号灰蓝，语言标签在块上方。
- Table: faux table，14px 字号，细线网格，表头浅蓝底，移动端不横向溢出。
- Divider: 支持细线、虚线和蓝色短线语义，但渲染器默认使用细线。

## Mobile Rules

- 内容最大宽度 640px，默认左右内边距 20px。
- 主要验收宽度为 390px 和 430px。
- 所有样式内联，无 `style` 标签、无 `class`、无外部 CSS。
- 表格使用 faux table，不输出原生 table。
- 代码块使用 `pre-wrap`，不得依赖横向滚动。
- 图片必须 `max-width: 100%`、`height: auto`、圆角 8px。
- 细线只服务层级，不得把正文变成表单式网格。

## Quality Gates

- `themes/techspec.json` 与本文档同步维护。
- `fixtures/techspec-style-system.md` 是唯一验收 fixture。
- `goldens/techspec-style.html` 必须由该 fixture 渲染得到。
- 修改主题后运行 JSON 校验、Markdown 合约、HTML verify、visual audit、`typeset.py --check`，并在 390px 与 430px 宽度下检查无横向溢出。
