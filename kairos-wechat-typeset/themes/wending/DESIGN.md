# 稳境白纸 DESIGN.md

## Theme Position

稳境白纸是根据附件《微信公众号文章组件规范》落地的移动端公众号正文主题。它服务自我成长、心理秩序、生活方式、轻方法论和慢阅读文章，把“不确定世界里的确定感”翻译成暖白纸面、重宋标题、灰度元信息、细线秩序和稳定移动端节奏。

本主题不是平台 UI 复刻。参考图中的手机状态栏、公众号顶部栏、底部工具栏和右侧规格页都属于说明语境；正文主题只落地微信公众号编辑器可控的正文表面：标题、导语、正文、章节标题、图片、图注、引用、代码、列表、任务清单、表格、分隔线和结尾收束。

## Visual Language

- 关键词：暖白纸、思源宋体、黑灰秩序、克制留白、细线分隔、暗色代码块、8px 图片圆角。
- 适合：个人成长、心理秩序、生活方式、读书笔记、轻方法论、公众号组件规范示例。
- 禁止：渐变、强阴影、多色强调、大面积卡片堆叠、圆角胶囊按钮、装饰图标、营销 CTA、深色整页背景。
- 页面气质：像一份严谨的移动端文章规范，也像一篇安静的公众号正文。

## Tokens

附件给出的字号和行高是硬约束：

- H1: 28px / 1.4, 思源宋体 CN Heavy, `#111111`。
- H2: 22px / 1.5, 思源宋体 CN Bold, `#111111`。
- H3: 18px / 1.6, 思源宋体 CN Medium, `#111111`。
- 正文: 16px / 1.75, 思源宋体 CN Regular, `#333333`。
- 辅助信息: 14px / 1.6, 思源宋体 CN Regular, `#888888`。
- 引用: 16px / 1.75, 思源宋体 CN Regular, `#666666`。

主色只允许一组灰棕 accent，用于章节编号、细线和极少数强调。背景保持接近参考图的暖白色，引用和洞察使用浅灰纸面，代码使用暗色块。

## Components

- H1: 28px 重宋标题，左对齐，不加边框，依靠字重和留白建立开篇重量。
- H2 numeric: 复刻参考图正文左侧章节样式：大号灰棕数字、斜向细线符号、22px 中文章节标题。编号和标题分两行，形成明确的移动端段落换章。
- H2 non-numeric: 22px 黑色标题，底部一条浅灰细线，用于规格类小节。
- H3-H6: 18px 黑色小标题，紧凑承接正文。
- Paragraph: 16px / 1.75，颜色 `#333333`，段距紧凑但不拥挤。
- Lead: 开篇导语，16px 灰黑色，保持正文语气，不做封面式大标题。
- Quote: 浅灰纸面、16px 灰字、左上角引号符号，贴近参考图的引用块。
- Callout: 复用引用纸面，前置小号中文标签，避免彩色提示框。
- Pullquote / Insight: 浅灰纸面里的安静摘句，文字不夸张放大。
- Figure: 图片宽度 100%，高度自适应，圆角 8px；图注 14px 灰色，左对齐。
- Lists: 无序列表使用黑色圆点，有序列表使用数字，任务清单使用 12px 方框，已完成为黑底白勾。
- Code: 暗色 8px 圆角代码块，14px 等宽字体，带行号。
- Table: 14px 细线网格，表头浅灰底，单列移动端内不横向溢出。
- Divider: 一条浅灰水平线，不使用装饰符号。
- Closing note: 灰色居中收束，不模拟平台底部工具栏。

## Mobile Rules

- 内容最大宽度 640px，默认内边距 20px，优先服务 390px 与 430px 手机宽度。
- 图片必须 `max-width: 100%`、`height: auto`、圆角 8px。
- 表格使用 faux table，不输出原生 table。
- 代码块使用 `pre-wrap`，不得依赖横向滚动。
- 所有样式内联，无 `style` 标签、无 class、无外部 CSS。
- 章节标题、引用和代码块之间不能叠加过多边框；细线只用于必要的层级和分隔。

## Quality Gates

- `themes/wending.json` 与本文档同步维护。
- `fixtures/wending-style-system.md` 是唯一验收 fixture。
- `goldens/wending-style.html` 必须由该 fixture 渲染得到。
- 新增或修改主题后运行 JSON 校验、Markdown 合约、HTML verify、visual audit、`typeset.py --check`，并在 390px 与 430px 宽度下检查无横向溢出。
