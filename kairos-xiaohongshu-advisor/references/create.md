# 创作流程

适用：用户要求直接写或改写内容。

## 第一步：类型匹配（先选型，再落笔）

根据账号阶段和目标，从 SKILL.md 的类型匹配表中选 1-2 种类型。
不同阶段配比不同——新号用干货硬打是低效的，
应先用共鸣/反差型降低关注门槛。
如果用户没说明账号阶段，先追问，不要默认。

## 第二步：钩子设计

本质是制造能被一句话描述的信息缺口，不是单纯"吸引眼球"。
三种高效缺口：

- 错误认知缺口："90% 的人 XX 都错在这一步"
- 身份缺口："如果你刚开始做 XX"
- 代价缺口："我浪费了 3 个月才发现"

## 第三步：内容结构

- 前 3 秒 / 第一行：制造信息缺口
- 一句话建立信任：真实背景（不要完美人设，要真实经历）
- 核心信息：3 点以内，清晰 > 全面
- 互动引导：低门槛问题
- 叙事弧：脆弱 + 复原力。只有脆弱没有复原力 = 卖惨，
  换来同情不等于信任；帮用户补上"走出来的方法"这一步

## 收尾

写完跑一遍三问自检（见 SKILL.md 共享资产），
不清楚的地方直接说"这里还不够"，标注出来让用户补充，
而不是硬写填满。
